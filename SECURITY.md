# Security Policy

## Supported Versions

The following versions are currently supported with security updates.

| Version | Supported |
|----------|-----------|
| Latest   | ✅ Yes |
| Older    | ❌ No |

## Reporting a Vulnerability

We take the security of StockPro Inventory Management System seriously.

If you discover a security vulnerability, please do not disclose it publicly.

Instead, report it by emailing:

**Email:** mdnawaz6052@gmail.com

Please include:
- A description of the vulnerability.
- Steps to reproduce the issue.
- The affected page or feature.
- Screenshots or proof of concept (if available).

## Response Timeline

- Initial response: Within 24–48 hours.
- Status update: Within 7 days.
- Security fix: As soon as possible based on severity.

## Security Best Practices

- Always use HTTPS.
- Keep dependencies up to date.
- Protect API keys and secrets.
- Never commit passwords or credentials to the repository.
- Enable Firebase Authentication and Firestore Security Rules.
- Validate and sanitize all user input.
- Use least-privilege access for administrators.
- Regularly review logs for suspicious activity.

## Disclosure Policy

Please give us reasonable time to investigate and fix the issue before making it public.

Thank you for helping keep StockPro Inventory Management System secure.
